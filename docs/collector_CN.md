# 任务汇总模块 (Task Hub Collector)

## 模块概述
`task-hub-collector` 模块负责系统的“复盘”。它汇总所有已关闭的任务，提取关键成果，为人类用户或主调度 Agent 提供直观的进度汇报。

## 核心功能
1.  **成果提取**: 遍历所有 `task/done` 状态的 Issue，提取执行者的汇报内容。
2.  **进度分析**: 计算任务完成率、阻塞点及整体进度。
3.  **生成报告**: 输出格式化的进度周报或日报。

## 工作流程
1.  **数据抓取**: 获取最近关闭的 Issue 列表及其评论。
2.  **结构化汇总**: 将零散的评论转化为结构化的成果清单。
3.  **归档管理**: (可选) 为已汇总的 Issue 打上 `archived` 标签。

## 报告模板示例
- **已完成任务**: [任务 A] - 由 Agent X 完成 - 成果链接
- **待处理任务**: [任务 B] - 状态: 阻塞 (缺失 API Key)

## 使用场景
主调度 Agent 定期运行此技能，以便及时向用户汇报“哥哥，任务都做完啦”。
