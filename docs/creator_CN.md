# 任务分配模块 (Task Hub Creator)

## 模块概述
`task-hub-creator` 模块是整个系统的“大脑”。它的主要职责是将复杂、模糊的宏观目标拆解为具体、可执行的微观任务，并将这些任务以 GitHub Issues 的形式派发给合适的执行者。

## 核心功能
1.  **任务拆解**: 接收用户指令，分析并拆解为多个独立子任务。
2.  **标准化输出**: 生成包含背景、指令、优先级和验收标准的标准化 Issue 模板。
3.  **精准派发**: 根据任务性质打上对应的执行者标签（如 `skill/answer`, `skill/taizi`）。

## 工作流程
1.  **输入分析**: 分析任务的复杂程度和依赖关系。
2.  **Issue 创建**: 使用 `gh issue create` 命令。
3.  **标签管理**: 自动添加 `task` 和 `priority/*` 标签。

## 标签规范
- `task`: 表示这是一个待处理的新任务。
- `priority/p0`: 紧急且重要，需立即执行。
- `skill/*`: 指定特定的 Agent 或技能组领取。

## 使用场景
当用户下达一个跨领域或多步骤的复杂指令时，主调度 Agent（如小溪）会调用此技能进行分工。
